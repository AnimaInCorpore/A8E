# ANTIC

> Hardware emulation reference: Before implementing any Atari 800 XL PAL machine related hardware emulation, use the [AHRM](/AHRM/index.md) as reference.

- Files: `A8E/Antic.c`, `A8E/Antic.h`, `A8E/AtariIo.c`, `A8E/AtariIo.h`, `A8E/tests/antic_timing_probe.c`
- Purpose: handle display list processing, DMA timing, and display NMIs.
- Status: updated on 2026-04-03 (`implemented`).
- Notes: register accessors are in `Antic.c`; scanline progression, display list fetch, DLI scheduling, and VBI timing are driven from the timed-event path in `AtariIo.c`. DLI deadlines are anchored to the in-line beam clock (`llCycle`), and the timed-event scheduler keeps separate beam/master wake thresholds so raster DLIs follow the playfield fetch position without regressing longer-running vertical timing or master-clock POKEY/SIO events. Per AHRM 4.8, ANTIC asserts NMI at cycle 8 on the last scanline of a DLI-enabled mode line; the NMI handler entry sequence begins at the first instruction boundary on cycle 10 or later. Same-scanline `NMIEN` writes now follow the AHRM gating window for DLIs: writes by cycle 7 enable the current interrupt, writes on cycle 8 can still suppress it, and an exact cycle-7 enable delays the CPU-visible DLI by one beam cycle while NMIST still latches immediately. VBI fires at the beginning of scan line 248 (AHRM 4.8). VCOUNT now reflects the next scanline starting on cycle 111 of the previous line per AHRM 4.10, including the one-cycle PAL end-of-frame anomaly (`$9C` on the final scanline's cycle 111 before wrapping to `$00`). WSYNC halts the CPU until cycle 105, with writes up to cycle 103 targeting the current line and writes at cycle 104 or later waiting for the next line (AHRM 4.9). JVB detection now ignores DLI/vertical-scroll bits (`$41/$C1/...` via masked decode), JVB+DLI replay re-arms one DLI per scanline until VBL, and NMIST DLI bit (bit 7 per AHRM 4.8) follows AHRM lifetime semantics (cleared at scan line 248 or `NMIRES`, not every scanline). Active-line playfield geometry now uses the corrected AHRM 4.14 width starts (clock 26/18/10 after HSYNC for narrow/normal/wide), and HSCROL-promoted windows delay from those baselines by one cycle per two color clocks of scroll. DMA steal positions match AHRM 4.14: refresh at cycles 25/29/33/37/41/45/49/53/57, display list instruction at cycle 1, LMS/jump address at cycles 6–7, missile at cycle 0, players at cycles 2–5 (AHRM 4.13). Playfield DMA is now scheduled per line cycle instead of being charged as a bulk line-start stall: first-row display/name fetches fill a 48-byte line buffer, repeated scanlines reuse those buffered bytes, character modes 2-7 place the character-data fetch in the later `+3` cycle slot, and late fetches after cycle 105 fall back to the virtual CPU bus / refresh-drop value instead of stalling the CPU. The native `antic_timing_probe` harness drives the real timed-event path and now also locks down the cycle-7/cycle-8 `NMIEN` edge cases alongside DLI cycle-8 assertion, VBI entry on line 248, the cycle-111 VCOUNT boundary, and JVB+DLI replay re-arming/stopping behavior.
- Issues: VBI-side `NMIEN` gating and `VSCROL`/DLI same-line deadlines are still not modeled.
