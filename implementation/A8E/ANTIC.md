# ANTIC

> Hardware emulation reference: Before implementing any Atari 800 XL PAL machine related hardware emulation, use the [AHRM](/AHRM/index.md) as reference.

- Files: `A8E/Antic.c`, `A8E/Antic.h`, `A8E/AtariIo.c`, `A8E/AtariIo.h`, `A8E/tests/antic_timing_probe.c`
- Purpose: handle display list processing, DMA timing, and display NMIs.
- Status: updated on 2026-04-08 (`implemented`).
- Notes: register accessors are in `Antic.c`; scanline progression, display list fetch, DLI scheduling, and VBI timing are driven from the timed-event path in `AtariIo.c`. DLI deadlines are anchored to the in-line beam clock (`llCycle`), and the timed-event scheduler keeps separate beam/master wake thresholds so raster DLIs follow the playfield fetch position without regressing longer-running vertical timing or master-clock POKEY/SIO events. Per AHRM 4.8, ANTIC asserts NMI at cycle 8 on the last scanline of a DLI-enabled mode line; the NMI handler entry sequence begins at the first instruction boundary on cycle 10 or later. Same-scanline `NMIEN` writes follow the AHRM gating window: writes by cycle 7 enable the current interrupt (with one-beam-cycle delay); writes on cycle 8 can still suppress it; NMIST latches unconditionally. VBI fires at the beginning of scan line 248 (AHRM 4.8). VCOUNT reflects the next scanline starting on cycle 111 of the previous line per AHRM 4.10, including the one-cycle PAL end-of-frame anomaly. WSYNC halts until cycle 105; writes ≤ cycle 103 target the current line, writes ≥ cycle 104 target the next line (AHRM 4.9). JVB detection ignores DLI/VSCROL bits; JVB+DLI replay re-arms one DLI per scanline until VBL; NMIST DLI bit follows AHRM lifetime (cleared at scan line 248 or NMIRES). Active-line geometry uses the corrected AHRM 4.14 width starts; HSCROL-promoted windows delay by one cycle per two color clocks. DMA steal positions match AHRM 4.14: refresh at cycles 25/29/33/37/41/45/49/53/57, DL instruction at cycle 1, LMS/jump address at cycles 6–7, missile at cycle 0, players at cycles 2–5 (AHRM 4.13). Playfield DMA is scheduled per line cycle; first-row fetches fill a 48-byte line buffer; character modes 2–7 place the character-data fetch in the `+3` cycle slot; late fetches after cycle 105 use the virtual CPU bus / refresh-drop value. Character-mode CHBASE and CHACTL are snapshotted before the render loop in all modes 2–7, so DLI writes to either register cannot affect the current scanline mid-render. CHACTL bits 0 (blank), 1 (invert), and 2 (vertical reflection) are now fully implemented in modes 2/3; bit 2 reflection is applied in modes 4–7 as well. The native `antic_timing_probe` harness locks down DLI cycle-8 assertion, VBI entry on line 248, the cycle-111 VCOUNT boundary, JVB+DLI replay, and the cycle-7/cycle-8 NMIEN edge cases.
- Issues: VBI-side `NMIEN` gating and `VSCROL`/DLI same-line deadlines are still not modeled. CHBASE is snapshotted once before the C render loop, so mid-scanline DLI writes to CHBASE are completely ignored (unlike real hardware where CHBASE can change mid-line with a 2-cycle delay per AHRM 4.4). The JS core already handles mid-scanline CHBASE correctly via the delayed-latch polling in `currentCharacterBaseRegister()`.
