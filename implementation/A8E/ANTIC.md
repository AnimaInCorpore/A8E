# ANTIC

> Hardware emulation reference: Before implementing any Atari 800 XL PAL machine related hardware emulation, use the [AHRM](/AHRM/index.md) as reference.

- Files: `A8E/Antic.c`, `A8E/Antic.h`, `A8E/AtariIo.c`, `A8E/AtariIo.h`
- Purpose: handle display list processing, DMA timing, and display NMIs.
- Status: verified on 2026-02-23 (`implemented`).
- Notes: register accessors are in `Antic.c`; scanline progression, display list fetch, DLI scheduling, and VBI timing are driven from the timed-event path in `AtariIo.c`. DLI deadlines are anchored to the in-line beam clock (`llCycle`), and the timed-event scheduler keeps separate beam/master wake thresholds so raster DLIs follow the playfield fetch position without regressing longer-running vertical timing or master-clock POKEY/SIO events. Per AHRM 4.8, ANTIC asserts NMI at cycle 8 on the last scanline of a DLI-enabled mode line; the NMI handler entry sequence begins at the first instruction boundary on cycle 10 or later. VBI fires at the beginning of scan line 248 (AHRM 4.8). VCOUNT reads reflect the next-scanline value from cycle 100 onward (AHRM 4.14 event timing chart). WSYNC halts the CPU until cycle 105, with writes up to cycle 103 targeting the current line and writes at cycle 104 or later waiting for the next line (AHRM 4.9). JVB detection now ignores DLI/vertical-scroll bits (`$41/$C1/...` via masked decode), JVB+DLI replay re-arms one DLI per scanline until VBL, and NMIST DLI bit (bit 7 per AHRM 4.8) follows AHRM lifetime semantics (cleared at scan line 248 or `NMIRES`, not every scanline). DMA steal positions match AHRM 4.14: refresh at cycles 25/29/33/37/41/45/49/53/57, display list instruction at cycle 1, LMS/jump address at cycles 6–7, missile at cycle 0, players at cycles 2–5 (AHRM 4.13).
- Issues: (1) `DLI_HORIZONTAL_OFFSET` is 14, not 8: AHRM 4.8 says ANTIC asserts NMI at cycle 8, but the emulator schedules the DLI event at cycle 14 from line start, approximately combining the NMI pin assertion (cycle 8), the 2-cycle CPU acknowledgment, and partial instruction-finish time into one offset. (2) VBI fires on scan line 249 instead of 248: AHRM 4.8 says VBI is asserted at the beginning of scan line 248; the code clears the DLI status bit on line 248 (correct) but delays the VBI NMI assertion to line 249 (one line late).
