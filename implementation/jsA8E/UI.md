# UI / Interface

- Files: `jsA8E/index.html`, `jsA8E/style.css`, `jsA8E/js/app/ui.js`, `jsA8E/js/app/hostfs_ui.js`, `jsA8E/js/app/assembler_ui.js`, `jsA8E/js/app/snapshot_ui.js`, `jsA8E/js/core/assembler_core.js`, `jsA8E/js/core/assembler/shared.js`, `jsA8E/js/core/assembler/lexer.js`, `jsA8E/js/core/assembler/preprocessor.js`, `jsA8E/js/core/assembler/parser.js`, `jsA8E/js/core/assembler/object_writer.js`, `jsA8E/js/core/assembler/assembler.js`, `jsA8E/js/app/a8e.js`, `jsA8E/js/app/version.js`
- Purpose: provide browser controls and status for emulator operation.
- Status: verified on 2026-03-09 (`implemented`).
- Notes: handles ROM/disk load, start/pause/reset, turbo/audio toggles, virtual keyboard/joystick, fullscreen interactions, and H: file-manager UI flows. `hostfsPanel` is now width-aligned with the keyboard and uses reduced panel/list minimum heights with moderate dynamic sizing so the default open state is smaller while empty-state readability is preserved. Visible UI naming uses `HostFS`, with the panel title showing `HostFS (H:)`. Status bar now reports DOS-style free space as `FREE SECTORS` (999 logical sectors at 128 bytes each, per-file sector rounding). The top bar now also exposes assembler and snapshot icons. The run/pause button now updates optimistically: it flips immediately on click based on the requested state, while the existing live state/debug updates reconcile the final running flag once the worker or main-thread runtime has actually processed the request. `btnSnapshots` toggles `snapshotPanel`, implemented in `js/app/snapshot_ui.js`, with one IndexedDB-backed quick-save slot plus export/import buttons, all routed through the same snapshot APIs used by automation instead of introducing a UI-only state path. The snapshot panel now follows the same vertical sizing pattern as the other sub-panels: it opens with a real minimum height and its body stretches to fill that space instead of collapsing to the compact card content. The top-bar assembler editor icon (`btnAssembler`) still toggles `assemblerPanel`; assembler source load/save and panel behavior remain in `js/app/assembler_ui.js`, while the assembler engine is available as `js/core/assembler_core.js` (`window.A8EAssemblerCore`) backed by internal modules (`js/core/assembler/{shared,lexer,preprocessor,parser,object_writer,assembler}.js`). UI assembly calls route directly to this core module (no duplicated in-UI assembler pipeline), and builds now pass `sourceName` plus a HostFS-backed include resolver so preprocessor `.include` can load files from H:. Current assembler output remains DOS-loadable `.XEX` files to H: and now includes preprocessor support for `.include`, `.define`/`.undef`, `.macro`/`.endmacro` (`.endm`), `.local`, and conditional assembly (`.if`/`.ifdef`/`.ifndef`/`.elseif`/`.else`/`.endif`) before normal assembly directives (`.SEGMENT`, `.IMPORT`, `.GLOBAL`, `.ORG`/`ORG`, `*=`, `.RUN`/`RUN`, `.DS`/`.RES`, `.BYTE`/`.DB`, `.WORD`/`.DW`/`.ADDR`, `.LOBYTES`, `.HIBYTES`, `.TEXT`, `.ASSERT`, `.ERROR`, `.END`/`END`, constants, labels, and common 6502 addressing modes). In current XEX mode, `.SEGMENT`, `.IMPORT`, and `.GLOBAL` are accepted as metadata/no-op directives while relocatable object emission is still pending. The disk file-picker path now uses the awaitable mount flow but does not auto-reset or auto-start, so it can be used for plain disk flipping. Drag-and-drop keeps the reboot/start behavior so dropping a new ATR/XEX onto the screen still boots it immediately. The assembler editor now includes a left breakpoint gutter with per-line toggles; successful assembly publishes `line -> CPU address` metadata so breakpoints can be enabled/disabled by source line and pushed to runtime dynamically. When execution is paused by a breakpoint/step action, debug controls (`Step`, `Step Over`, `Continue`) are shown in the assembler toolbar, and the normal assembler `Run` button is hidden until `Continue` resumes execution. The assembler bottom status line appends live debug registers (`PC/A/X/Y/SP/P`) and shows breakpoint-hit context (including mapped source line when available). `js/app/automation_api.js` now exposes a grouped browser-facing automation facade as `window.A8EAutomation`; `ui.js` attaches the live app instance so external tools can discover capabilities, query full system state, wait on pause/time/cycle conditions, load ROMs/disks, control execution, inject keyboard/joystick/console input, inspect memory and disassembly, access HostFS, assemble source, run XEX payloads, and request screenshot, artifact capture, or snapshot operations without reaching into internal UI state. `window.A8EUI.getApp()` remains available as a low-level escape hatch, but the intended public entrypoint is the automation facade. Sub-panel styling (HostFS, Assembler, Snapshot) now uses shared CSS custom properties (`--sub-panel-*`) for colors, borders, and radii, ensuring consistent button sizes, toolbar/statusbar spacing, scrollbar appearance, and disabled-state opacity across all panels. These panels follow the same structural convention: titlebar is purely informational (icon + title text), toolbar contains all action buttons organized into labeled groups. The HostFS toolbar uses two groups (Add: Upload, Folder | Select: All, None, Download, Delete). The assembler toolbar uses two logical groups (File: dropdown + load + name + save | Build: assemble + run). The snapshot toolbar uses two logical groups (Quick Slot: Save, Load | File: Export, Import). `Run` auto-saves source before assembling. Output `.XEX` filename is auto-derived from the source name. Keyboard shortcuts: `Ctrl+S` save, `Ctrl+Shift+B` assemble, `Ctrl+Enter` run/continue (when debugger controls are visible).
- Issues: HostFS intentionally uses a flat basename-only namespace (`file.name`) for H: DOS/FMS-style workflows; directory hierarchy is not preserved and duplicate filenames from different subfolders can collide/overwrite.
