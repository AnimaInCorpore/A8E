## 5.9 Paddle scan

POKEY has inputs for eight potentiometer ('pot') inputs, typically used to read paddles. Pulse timing is automatically handled in hardware, relieving the CPU of the need to poll the inputs and simply requiring values to be read after the polling has completed.

## Polling mechanism

The pot inputs are designed to be connected to a capacitor charged up by a variable voltage, where the voltage is determined by a pot-based divider in the paddle controller. The voltage from the controller determines the rate at which the capacitor charges up, which in turn changes the time taken for the capacitor to charge up to threshold voltage. POKEY measures how long this takes and reports this as the controller position. Afterward, dumping transistors drain the capacitors and reset them for the next read.

The polling process is started by a write to the POTGO register. This resets the main counter, which begins counting up from 0 once per scan line. The value of this counter is continuously latched into each pot position register (POT0-7) until the corresponding input reaches threshold. The counter stops at 228, which is latched into the registers for any pot inputs that have not yet reached threshold.

Normally, the polling counter is driven by the same clock that is used by the keyboard scan. Therefore, paddle scanning is inoperative when initialization mode is active and the keyboard clock is frozen. The exception is when fast pot scan mode is enabled, which selects the machine clock instead of the 15KHz clock.

## Polling status

The ALLPOT register can be used to check the state of each input. Each bit in ALLPOT corresponds to one of the inputs and reads 1 while the input is being polled, then 0 when polling is complete. Therefore, the amount of time that each bit stays 1 before clearing is proportional to the count eventually read. ALLPOT is forced to $00 once the scan has finished, regardless of the state of the inputs.

ALLPOT also indicates which POTn registers are being updated continuously from the main counter. If an input somehow dips back below input threshold during the scan, the corresponding bit in ALLPOT will revert to 1 and the POTn register will resume updating from the main counter.

ALLPOT is not clocked by the 15KHz scan counter. It is updated on every cycle and can change at any time regardless of the scan counter clock timing. This also means that the POT0-7 registers can stop counting and latch their final values at any time, not only every 114 cycles.

## Fast pot scan mode

Setting bit 2 of SKCTL enables fast pot scan mode, which switches the counter from the 15.7KHz keyboard clock to the 1.79MHz machine clock. Writing POTGO is still required to reset the counters and start the scan; changing SKCTL alone does not begin one. The scan time is shortened to two scan lines, and the values latched into the POT0-7 counters are proportional to clock cycles instead of scan lines. The counter stops one value higher in this mode than in normal scan mode (229 vs. 228).

Fast pot scan mode cannot be used to scan regular paddles. Although it technically functions, it is practically useless as the counts are 114 times higher, giving 0-229 at the extreme low end of the normal pot value range. This is because the rate at which the paddles charge the input capacitors is far too slow for the faster scan rate.

A side effect of enabling fast pot scan mode is that it also disables the capacitor dump transistors. This means that, by default, simply enabling fast pot scan mode will not work because the capacitors will never be drained below threshold voltage after they have charged up, no matter how many times POTGO is strobed. Inputs that have already reached input threshold at the start of the scan will not have their position registers updated at all. The dump transistors are disabled as soon as fast pot scan mode is enabled, so attempting to enable fast pot scan mode and then write POTGO can fail when the capacitors charge up past the threshold in between the two writes. The dump transistors are on whenever slow pot scan mode is enabled and a scan is not occurring.

## Reading live counters

The POT0-7 registers are continuously updated from the main counter during the scan while the corresponding input is below threshold (ALLPOT bit is set), during which time reading those registers will reflect the live updating count. However, the value returned can be non-monotonic due to a conflict between the register read and the increment of the main counter. This is apparently due to the count being sampled when it is unstable, due to differing rise and fall times when the counter increments. The result is the bitwise AND of two adjacent values, with zero bits for any bit that started as a zero before the increment or ends as a zero after it.

For slow pot scan, this transitional value is seen for a single cycle every 114 cycles when the counter is advanced by the 15KHz clock, and shortly before the scan ends.

For fast pot scan, transitional values are seen on every cycle while the pot scan is active. Starting after the write to POTGO, the following values can be expected:

$00, $00, $00, $02, $00, $04, $04, $06, $00, $08, $08, $0A, $08, $0C, $0C, $0E, $00

Although the initial few cycles are not observable on a stock system, this is consistent with the counter being cleared during the cycle after the write to POTGO, and the incrementing on each cycle after that. Thus, the second cycle after the write to POTGO reads:

- $00 AND $01 = $00
- $01 AND $02 = $00
- $02 AND $03 = $02
- $03 AND $04 = $00
- etc.

The effect is that the counter that is still updating cannot be read stably. This is true even in slow pot scan mode, where the counter is mostly stable, but can still return a transitional value in 1 out of every 114 cycles.

Though this pattern does appear to be caused by timing issues in the logic, the effect is fully deterministic and the same values are produced on each cycle.

Once a pot input rises above threshold, the counter stops updating and this effect no longer occurs for that counter. In particular, it is possible for an odd value to appear even though the transitional values seen while the counter is updating are always even. Therefore, while reading the live counter can produce transitional values, the value stored into the POT0-7 registers does not show that effect.

## Scan timing

In fast pot scan mode, the scan will end and the ALLPOT register is forced to $00 starting with 231 cycles after the write to POTGO. Although the scan stops at a count of 228, it takes one cycle for this to be detected and for the scan to stop. In fast scan mode, this results in the counter over-running to 229 ($E5) and that value being latched into any still updating counters with the corresponding bit in ALLPOT still set. ALLPOT then reads as $00 on the following cycle with counters unchanged.

In slow scan mode, active counters initially show $00 and a transitional value of $00 when the main counter increments for the first time. This happens on the next 15KHz clock tick, with the increment cycle happening 81+114N cycles after the write to SKCTL that exits initialization mode. The counters then show $01 after this first increment cycle. 227 × 114 = 25,878 cycles after this initial increment, the final increment from 227 to 228 will occur with a transitional value of $E0, any remaining active counters will show a stable $E4 on the cycle after that, and one cycle later ALLPOT reads $00.

## Truncated scans

A full paddle scan takes 228 scan lines or cycles, depending on the mode. The entire time is required regardless of when the individual pot scan counters latch. If POTGO is retriggered before the previous scan finishes, the dump transistors will not have had a chance to drain the capacitors before the next scan starts, resulting in truncated counts. For instance, restarting the scan after 64 scan lines will result in counts 64 lower for any inputs that have not reached threshold yet, since the main counter will restart from zero after the capacitors have already charged up by 64 counts worth of time. Any pot inputs that are already over the threshold when the scan is restarted will not update at all. The newly triggered scan will run for a full 228 counts as usual unless it too is interrupted.

This behavior makes it possible to read the entire range of a CX-30 paddle controller by extending the scan to longer than its usual duration.

## Suspended scans

In slow scan mode, the pot scan can be suspended when initialization mode is entered by clearing SKCTL[1:0], since that stops the 15KHz clock. When this happens, the main scan counter stops incrementing and the scan is suspended indefinitely until either initialization mode is exited to restart the 15KHz clock or fast pot scan mode is enabled. During this time, the capacitors will continue to charge, and counters may latch if inputs rise above threshold, but the latched values will be constant since the main counter is not counting. The result is that when the scan resumes, any counters that latch afterwards will receive lower counts due to the counts missed while the scan was suspended.

As fast scan mode uses the machine clock instead of the 15KHz to drive the scan counter, it is unaffected by initialization mode.

## Discharge time

For scans to return regular values, the paddle capacitors must be sufficiently discharged between scans to avoid the resulting values being biased by residual charge in the capacitors. By default, dump transistors are enabled whenever slow pot scan mode is selected and no pot scan is occurring.

The dump transistors can initially drain the capacitors with an exponential decay starting at an approximate rate of 2V per 10µs, draining from 4.6V to 0.2V after 40µs and 0.1V after 50µs. Thus, about 100 cycles with the dump transistors on is enough to ensure no meaningful error in the next pot scan.

When paddles are rotated to the far right (clockwise), the resistance is low enough and enough current supplied that the dump transistors are noticeably unable to fully drain the capacitors. This happens with positions that produce slow pot counts of ~13 or lower. At the farthest position, the dump transistors are only able to drain the capacitors down to approximately 0.4V. In theory, this can produce an error in the next scan of as much as ~2ms (30 counts) in the extreme case of a minimum value immediately followed by a maximum value, but in practice this effect is negligible due to physical limits on how fast the paddle knob can be rotated between scans.

## Overcharge

The Schmitt triggers on POKEY's pot inputs activate with a threshold of about 2.1V, upon which the associated counter stops updating. However, the capacitors can still continue to charge up to about 4.8V. This is of no consequence with normal scan usage as the dump transistors can easily empty this extra charge within dozens of cycles, but it is relevant when insufficient time passes between scans for this to occur.

## Reset

Other than the stopping of the slow pot scan clock, the pot scan logic is not affected by initialization mode. It is possible for POKEY to run an abnormal pot scan of up to 256 counts instead of 228 on power-up depending on the power-up state of the counter, but this is always cleared the first time POTGO is written.

Note that the traditional pattern of clearing all POKEY registers will also start a paddle scan, which by default will take 228 scan lines to finish. The first intentional paddle scan afterward may be triggered earlier than this, causing it to be abnormal unless enough time has passed for this 'reset scan' to finish and then for the dump transistors to empty the capacitors. The needed delay can be shortened by temporarily enabling fast pot scan mode to force the scan to finish within 228 cycles.

## Unconnected pot inputs

If no controller inputs are connected to the paddle input lines and inputs are floating, no appreciable charging or discharging of the related paddle capacitors occurs unless the dump transistors are turned on. When those are disabled by turning on fast pot scan mode, the charge can persist on the capacitors for a very long time enough to hold the pot inputs above or below threshold for minutes or longer, effectively indefinitely. This effect is observable by reading ALLPOT during a fast pot scan.

## Grounded pot inputs

The normal operation for a pot input is for a paddle to charge the paddle capacitor through its variable resistance during a scan, and then for the capacitor to be discharged outside of scans by the dump transistors. However, it is also possible for the controller to discharge the capacitors. The CX-21/23/50 Keyboard Controllers in particular do this when a button is pressed in one of the left two columns.

The main difference with a grounded input is that it can be read in fast pot mode without having to periodically switch to slow pot mode to drain the capacitors. This allows simpler and faster reading of digital inputs through the pot inputs. With a paddle or other controller that only sources current, the capacitor will eventually charge above threshold, at which point the POT register will stop updating and the ALLPOT bit will always read 0. However, with the Keyboard Controller able to both ground and pull up the input, ALLPOT can be used to read the digital state during a fast scan. The POT register still can't be used reliably as it won't update if the controller input changes from a 0 to 1 outside of the fast scan.

## Internal inputs

On the XL/XE computers and the XEGS, which only have two joystick ports, pot inputs 4-7 are hardwired to ground and will always read 228 (normal scan mode) or 229 (fast scan mode).
