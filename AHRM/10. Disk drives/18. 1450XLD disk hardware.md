## 10.18 1450XLD disk hardware

The main controller in the 1450XLD PDD is an 8040 controller running at 10MHz. It is connected to a 2797 FDC, 4K of firmware ROM, and 256 bytes of external RAM for the sector buffer (in addition to the 256 bytes of internal RAM in the 8040).

## Transfer interface

The transfer interface between the disk controller and the PBI handler consists of a pair of BUSY and ATTN bits and a pair of read and write data latches. This facilitates byte-oriented transfers.

From the computer side, $D110-D113 doubles as both the write latch and the read strobe, while $D114-D117 is the read latch. Writes to the controller are done by writing to $D110, which also sets the BUSY flag; $D1FF bit 0 can then be polled to detect when the BUSY flag is reset again by the controller. Reads from the controller are done by writing a dummy byte to $D110, waiting for BUSY to clear, and then reading the byte from $D114. BUSY can only be set by the computer and cleared by the controller, and while it is exposed in $D1FF, it cannot trigger an IRQ.

## Transfer protocol

The protocol is essentially the SIO command/data frame protocol. ATTN is used as the analogue of the SIO command line, and is borrowed from an unusual place: bit 7 of $D108-D10B in of the modem control latch in the PBI device 1. The command frame is initiated by writing a byte to set BUSY, asserting ATTN (setting bit 7), waiting for BUSY to clear, then clearing ATTN. Afterward, the standard 5-byte command frame including SIO checksum is written. Note that ATTN is not asserted during the entire command frame as the SIO command line signal is.

This is in turn followed by the traditional command ACK/NAK, optional write data frame + checksum + data ACK/NAK, Complete/Error, and read data frame + checksum.

This protocol is somewhat prone to desync due to the way that the controller monitors the ATTN line and the random dummy bytes that can be written by the PBI handler when reading bytes. To combat this, the controller ignores $00 bytes at the start of a command frame, waiting until the first non-$00 byte before counting off 5 bytes.

The controller times out transfers after about 10ms. This is per byte transfer and is reset with each successively transferred byte.

## 8040 I/O map (non-TONG)

Table 70 lists the I/O pin assignments for the 8040 controller on the non-TONG version of the disk controller, used with the 1450XLD with short motherboard and the parallel disk drive on a side board. 77

Table 70: 1450XLD non-TONG controller I/O map

| Unit   | Pin   | Direction   | Usage                                                |
|--------|-------|-------------|------------------------------------------------------|
| Port 1 | P17   | Output      | Controller busy clear (on falling edge)              |
| Port 1 | P16   | Output      | RAM I/O enable (1 = I/O, 0 = memory)                 |
| Port 1 | P15   | Output      | FDC/RAM select (1 = RAM, 0 = FDC)                    |
| Port 1 | P14   | Output      | FDC density select (0 = FM, 1 = MFM)                 |
| Port 1 | P13   | Output      | FDC/RAM enable (1 = enable either, 0 = disable both) |
| Port 1 | P12   | Output      | FDC write precompensation enable (1 = enabled)       |
| Port 1 | P11   | Output      | FDC register select (A1)                             |
| Port 1 | P10   | Output      | FDC register select (A0)                             |
| Port 2 | P27   | Input       | unused                                               |
| Port 2 | P26   | Output      | RAM reset (1 = reset)                                |
| Port 2 | P25   | Output      | Write latch bus enable (1 = enabled)                 |
| Port 2 | P24   | Output      | Drive motor enable (1 = on)                          |
| Port 2 | P23   | Output      | ROMA11                                               |
| Port 2 | P22   | Output      | ROMA10                                               |
| Port 2 | P21   | Output      | ROMA9                                                |
| Port 2 | P20   | Output      | ROMA8                                                |
| Misc   | T0    | Input       | BUSY sense                                           |
| Misc   | T1    | Input       | ATTN sense                                           |
| Misc   | /INT  | Input       | FDC DRQ                                              |

P17 and T0 are used to manage the flip/flop that controls the BUSY signal. T0 senses when it is set by a byte being written from the computer, and P17 is momentarily pulled low to clear it. T1 senses the ATTN output, which is the equivalent of SIO COMMAND.

P15, P16, and P25 select between the FDC, RAM, and write latch. P25=1 enables reading latched data written from the computer, while P25=0 enables FDC or RAM access. For the latter, P15 selects between the FDC and the RAM.

P16 further selects between the I/O and memory halves of the 256-byte 8155H RAM chip, which also contains timer and bidirectional I/O port logic. Port B lines 0-3 are used for drive selection purposes, with a D0=1 selecting drive 0. The known firmware for this hardware only supports drive 0.

The read latch, which holds data from the 8040 to be read by the computer, doesn't have an enable. It simply captures all bytes written to the bus by the 8040 even if they are intended for the RAM or FDC.

The DRQ line is connected to the /INT interrupt input for fast checking during disk transfers. No inverter is used, so /INT is low and the JNI instruction branches when the FDC is not requesting a transfer.

Known firmware for this controller has CRC32 [7773B7DD] with size 4K and a date label of 3-12-84.

The TONG controller, used with the 1450XLD that has an integrated disk controller on a long motherboard, differs from the non-TONG controller in a minor but incompatible way. The 8155H is replaced with a plain 256 byte RAM, so no RAM I/O select or reset is needed and the drive select is instead driven by the top two bits of port 2.

Known firmware for this controller has CRC32 [03930886] with size 4K.
