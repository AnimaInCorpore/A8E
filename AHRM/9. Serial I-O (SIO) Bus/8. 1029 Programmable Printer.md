## 9.8 1029 Programmable Printer

The 1029 is a dot-matrix printer capable of printing 80 columns on regular 8.5' paper. In addition to regular text output, it also has bitmap graphics capability.

## SIO device ID

The 1029 responds to SIO device IDs $40 (P: or P1:) and $45 (P6:). All other printer device IDs are ignored.

## Write command

Write ($57) is used to send character data to the printer. The data frame size is 40 bytes and may include a complete or partial line. If an EOL ($9B) is present in the buffer, the line is ended at the first EOL and bytes after the EOL are ignored. Otherwise, the printer processes the characters in the data frame, buffering characters in an internal line buffer and printing a line if it is full. Thus, multiple Write commands are needed to build up an 80 character line.

The 820 mode byte in AUX1 is ignored by the 1029. It will accept Write commands with any mode byte, but if it receives the sideways ($53) mode, it still expects a 40 byte data frame. This leads to 138 (timeout) errors from CIO if sideways mode is enabled on the P: handler since in that case the handler tries to send a 29 byte data frame.

When a line is full or ended with EOL, the printer prints the line before sending the Complete ($45) code to the computer, after the line is printed but without waiting for the head to finish returning to the left.

The 1029 lacks a paper sensor, and will not return Error ($45) for an out of paper condition or any other condition. The only way a Write command fails is via a NAK ($4E) if a data frame checksum error is detected.

## Line overflow

As with the 1025, the 1029 prints a line as soon as its line buffer is full. It also avoids printing a blank line if an EOL is received after an exactly full line is printed.

## Status command

The status command ($53) returns a canned status response of $00, $00, $14, $29 which never changes. The printer does not even support the invalid command/data frame bits in the first byte.

The 1029 clears its line buffer and resets to default state when the status command is processed. This includes disabling underlining, international characters, elongation, bit image mode, and reverting to 6 LPI spacing.

## Escape processing

Special modes are selected in the 1029 through escape codes, which consist of the ESC character ($1B) followed by a character selecting the escape code. Unlike the 1025, all escape codes can be issued anywhere in a line, and more mid-line mode changes are supported. However, the 1029 does not allow bit 7 to be set on the escape code and will not recognize escape sequences using them.

Repeated ESC characters are treated the same as a single ESC character, so ESC ESC A is treated the same as ESC A. Unrecognized escape sequences are ignored and eat the unrecognized escape code character. ESC EOL is ignored but still treated as end of line.

Escape sequences do not have to be contained within a single Write command and can be split across commands.

## Dot matrix

Characters are printed within a 5x7 dot matrix. The baseline is at the bottom and there is no descender space, so characters like g and q are raised. There is no half-column spacing as in the 1025. A blank column separates characters in the same line.

As the printer is specified to print 80 characters across at 10 characters per inch and back to back lines at 9 lines per inch, dots are spaced approximately 0.0167' apart horizontally and 0.0159' vertically.

## International character mode

The normal 1029 character set consists of the ASCII characters $20-7F, with $7F (DEL) producing a solid block. ATASCII characters that differ in this range are not supported; Home ($7B), for instance, prints {. Bit 7 is ignored for characters besides ESC/EOL and the control characters $00-1A / $1C-1F are ignored without printing anything.

ESC CTRL+W switches to international character mode, enabling characters $00-1A, $1C-1F and replacing $60, $7B, and $7D-7F with the corresponding characters from the international ATASCII character set. ESC CTRL+X switches back to normal character mode. International character mode can be toggled on and off mid-line and both international and normal characters mixed within the same line.

## Line spacing

By default, lines are printed with a spacing of 6 lines per inch (LPI), which results in about a 3.5 dot row high gap in between lines. ESC 9 switches from 6 LPI to 9 LPI spacing, which places lines vertically adjacent to each other. ESC 6 changes back to 6 LPI spacing.

Unlike the 1025, line spacing can be changed during the middle of a line. The line spacing mode that is active when the line is ended or filled is the one that takes effect. If exactly 80 characters are printed at 6 LPI followed by ESC 9, the 80 characters are printed, a 6 LPI advance occurs, and then the spacing is switched to 9 LPI for the next line.

## Elongated mode

ESC CTRL+N switches from normal 10 CPI characters to 5 CPI elongated characters. In this mode, the column patterns for each character are doubled up horizontally and each character takes two character columns. ESC CTRL+O switches back to normal width characters. Unlike the 1025, the 1029 can switch to elongated characters in the middle of a line. Elongation persists across lines until turned off.

Although elongated characters take two character positions, they can be started at both even and odd positions. If an elongated character is printed at column 79 where there is not enough space for one, the line is ended and the elongated character is pushed to the next line.

## Underlining

ESC CTRL+Y enables underlining for characters. The underline is printed as an additional row of solid dots two rows beneath the main line, with one line of blank dots in between and roughly between rows at 6 LPI. The printer does this by doing a 9 LPI advance, printing the underlines at the 2 nd row of the next line, then advancing the paper by the remaining distance. Underlining persists until turned off again with ESC CTRL+Z or EOL is reached; underlining will stay enabled when printing lines longer than 80 characters.

For each character underlined, the 5 dots underneath the 5x7 cell and the 6 th dot to the right are printed. The underline is therefore continuous for adjacent characters. All characters are underlined including spaces, but the underline stops at an EOL.

Underlining is intended to be used with 6 LPI spacing. If it is enabled with 9 LPI spacing, the underline overlaps with the next line printed.

## Bit image mode

ESC CTRL+A enters bit image mode for graphics output. In this mode, the control bytes from the computer directly drive the seven dot columns of the printer, allowing full control over the printer's output.

The ESC CTRL+A sequence is directly followed by two bytes giving the length of the bitmap data in bytes as a 16-bit binary number, MSB first. Graphics data then follows as a byte per column, with bit 0 controlling the bottom dot and bit 6 controlling the top dot; bit 7 is ignored and ESC/EOL processing is suspended. The printer switches back out of bit image mode and resumes interpreting bytes as characters once the specified number of bit image bytes is processed. A full line requires 480 bytes of bitmap data.

For contiguous graphics taller than one line, 9 LPI mode must be selected to place lines vertically adjacent to each other. This is not automatic for bit image graphics and by default the printer will use the normal 6 LPI line spacing, which produces a gap in between.

Bit image mode may be started and stopped mid-line, mixing text and graphics. This can result in text characters being printed out of their normal alignment, allowing fine horizontal positioning without needing to render the text itself as graphics.

If the length value specified in the ESC CTRL+A sequence exceeds the remaining number of columns in the line, it is effectively clamped as the printer ends bit image mode anyway at the end of the line, and then interprets the remaining data as text. A length value of 0 is valid and is effectively a no-op.

Underlining is suspended during bit image graphics. If bit image graphics are started when underlining is enabled, the state is preserved and the underlining resumed for text printed after exiting bit image mode, but the bit image graphics themselves will not have an underline.

## Line wrapping

The printer rasterizes text or graphics into an internal buffer until either 480 columns are filled or an EOL is processed, then prints the line and resets the line buffer. For text, this is equivalent to a limit of 80 characters (normal) or 40 characters (elongated). However, the limit is more complex when graphics are involved.

Bit image graphics will fill as many columns as are provided or remain in the line, whichever is shorter. A maximum of 480 columns can fit within a line. If the line is filled before the image length is reached, bit image mode is forcibly disabled early and the remaining characters are processed as text instead of graphics.

A text character takes 5 columns followed by 1 column of spacing in normal mode, or 10 + 2 columns in elongated mode. If the character does not completely fit, the line is printed and the character put at the start of the next line. If the character fits within the line but the spacing doesn't, the character is still put within the line and the extra spacing extends into the right margin. This means that one column of bit image graphics can be printed followed by 80 characters within one line. However, printing 80 characters + 1 column bit image results in line being flushed after 80 characters, followed by the bit image being put at the beginning of the next line.

There is an anomaly if underlining is enabled and the spacing after a character overlaps the right margin. In this case, the underlining will be drawn under the following spacing as usual, even if it extends beyond the 480 th column. For instance, printing 1 column of bit image graphics followed by 80 underlined characters will result in the underline being drawn into the 481 st column. Elongated characters can extend the underline to the 482 nd column.
